# Named Heuristics, Tests & Canonical Phrases

## Diagnostic Tests

| Test | Pillar | How to Apply | What It Reveals |
|------|--------|-------------|-----------------|
| **Blur Test** | C, E | Gaussian blur to 5px. Can you identify the primary CTA? | Hierarchy problems — if the primary action disappears, emphasis has failed |
| **Barstool Test** | C | Read copy aloud as if explaining to a friend at a bar | Tone problems — if it sounds robotic, it needs conversational rewriting |
| **Copy Swap Test** | C | Replace your brand name with a competitor's. Does it still fit? | Specificity problems — if it works for any brand, it's too generic |
| **Write-With-Eraser** | C | Draft the copy, then delete 50%. Does meaning survive? | Density problems — surviving words are the essential copy |
| **Foggy Glasses Test** | E | Squint at the screen. What does your eye land on first? | Emphasis calibration — first-noticed should be most-important |
| **Relative Emphasis Check** | E | Cover the primary element. Is there a clear #2? Cover that. Is there a #3? | Hierarchy depth — if it collapses after one removal, emphasis is fragile |
| **30-Second Reward Test** | R | Use the interface for 30 seconds. Did you get positive feedback? | Reward deficit — no positive feedback = no reason to return |
| **F-Pattern Scan** | L, E | Trace an F across the screen. Does critical content hit the F's bars? | Placement problems — content outside the F is effectively invisible |

## Named Heuristics

| Heuristic | Source | Meaning |
|-----------|--------|---------|
| **Every Pixel Has To Be Earned** | growth.design | If an element doesn't help understand, decide, or act → remove it |
| **Customer Superpower Model** | Copywriting | Frame the benefit as a power the user gains, not a feature the product has |
| **Feature-to-Benefit Translation** | Copywriting | Feature = what it does. Benefit = what it means for the user. Most repeatable copy skill. |
| **Consider The Context** | Emphasis | Emphasis is relative. A red button on a red background = zero emphasis. |
| **Chart Type = Semantic Meaning** | Emphasis/Layout | Wrong chart type misleads even with correct data. Donut = progress. Pie = proportion. |
| **Detractors vs. General Activities** | Layout (Scorecard) | Semantic grouping principle for dashboard zones. Separate "problems" from "overview." |
| **Color Semantic Contracts** | Accessibility | When a color consistently means something, changing that mapping breaks trust. |
| **Forgiving Design** | Accessibility | Interfaces that work even when users make mistakes. Large targets, undo, confirmation. |
