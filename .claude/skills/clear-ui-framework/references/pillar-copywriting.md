# Pillar 1: Copywriting (C)

> "Layout makes a screen understandable. But Copywriting decides whether it's worth understanding."

## Core Thesis
Every word in a UI is a decision. Bad copy is too long, too generic, or unnecessary. Good copy earns its place by being specific, benefit-oriented, and action-driving.

## The Three Core Mistakes

### Mistake 1: Too Long
Users are cognitive misers (Fiske & Taylor, 1991). They conserve mental energy by default. Every additional word increases the chance they'll skip the entire block.

**Fix:** Write with an eraser. Draft the copy, then cut 50%. If the meaning survives, the cut words were noise.

### Mistake 2: Too Generic
Generic copy could belong to any product. It fails the Copy Swap Test — swap in a competitor's name and the sentence still works.

**Fix:** Use Voice of Customer (VoC) language. Pull from support tickets, reviews, and user interviews. Specific words from real users outperform marketing abstractions.

### Mistake 3: Unnecessary & Duplicate
Duplicate information (same message in title + subtitle + tooltip) doesn't reinforce — it clutters. Information that doesn't help the user understand, decide, or act is competing with what matters.

**Fix:** Apply the "Every Pixel Earned" principle. For each text element, ask: "Does this help the user understand, decide, or act?" If no, remove it.

## Four Practical Tips

### Tip 1: Emphasize Benefits Over Features
Features describe what the product does. Benefits describe what the user gets.

- Feature: "256-bit encryption" → Benefit: "Your data stays private"
- Feature: "Real-time sync" → Benefit: "Always up to date, no refresh needed"

**Model:** Customer Superpower — frame the benefit as a power the user gains.

### Tip 2: Reassure Proactively
Users carry anxieties they won't voice (hidden costs, data privacy, commitment). Address them before they become objections.

- "No credit card required" (addresses payment anxiety)
- "Cancel anytime" (addresses lock-in fear)
- "Your data is never shared" (addresses privacy concern)

**Named concept: Proactive Reassurance** — preemptively neutralize the objection the user hasn't articulated yet.

### Tip 3: Use Specific Words & Actions
Replace vague verbs with concrete ones. Replace abstract labels with descriptive ones.

- "Submit" → "Place your order"
- "Get started" → "Create your first project"
- "Learn more" → "See pricing details"

**Named concept: Feedforward** — the CTA label tells the user what will happen next, reducing uncertainty.

### Tip 4: Write Like You Talk
Apply the Barstool Test. If you wouldn't say it aloud to a friend, rewrite it. Conversational copy builds trust; corporate copy builds walls.

**Named concept: WIIFM (What's In It For Me)** — every screen element must answer the user's implicit question: "Why should I care?"

## Diagnostic Tests

| Test | How to Apply | What It Reveals |
|------|-------------|-----------------|
| **Blur Test** | Blur screen to 5px gaussian. Can you identify the primary CTA? | Hierarchy problems |
| **Barstool Test** | Read the copy aloud. Does it sound natural? | Tone problems |
| **Copy Swap Test** | Replace brand name with competitor. Does it still fit? | Specificity problems |
| **Write-With-Eraser** | Delete 50% of the words. Does meaning survive? | Density problems |

## Named Psychology Concepts

- **Cognitive Miser** (Fiske & Taylor, 1991) — people default to the least effortful interpretation. Don't make them work.
- **WIIFM** — implicit user question driving all attention allocation.
- **Voice of Customer (VoC)** — using the user's own language mirrors their mental model, reducing cognitive translation cost.
- **Feedforward** (Vermeulen et al., 2013) — a CTA that previews its outcome. "Start free trial" is feedforward; "Submit" is not.
- **Feature-to-Benefit Translation** — the single most repeatable copywriting skill. Feature = what it does. Benefit = what it means for the user.
- **Proactive Reassurance** — addressing objections before they surface. The copy equivalent of "I know what you're thinking..."
- **Customer Superpower Model** — frame the user as the hero who gains a new ability. "Track competitors in real time" > "Competitor tracking feature."
