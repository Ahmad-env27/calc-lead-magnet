# CLEAR Scorecard — Mechanics & Rubric

## Scoring System

5 pillars × 5 points each = 25 maximum.

The scorecard is NOT a grade. It's a **redesign priority map**. The goal is never precision — it's identifying gaps and enabling productive critique.

## Pillar Scoring Questions

### Copywriting (C) — Score 1–5

1. **Necessary?** — Is every piece of text earning its place? Could any be removed without losing meaning?
2. **Specific?** — Are labels unambiguous? Would a new user understand them without context?
3. **Benefit-oriented?** — Do CTAs and descriptions tell the user what they'll GET, not just what the feature DOES?
4. **Conversational?** — Does it pass the Barstool Test?
5. **Reassuring?** — Are user anxieties proactively addressed?

### Layout (L) — Score 1–5

1. **Grouped?** — Are related elements visually grouped (Proximity)?
2. **Consistent?** — Do same-role elements look the same (Similarity)?
3. **Simple?** — Is the layout free of unnecessary borders, dividers, and decoration (Simplicity)?
4. **Aligned?** — Do edges and baselines line up on a grid (Alignment)?
5. **Scannable?** — Can the user find what they need in 2 seconds (Continuity + F-pattern)?

### Emphasis (E) — Score 1–5

1. **Clear #1?** — Is there one unmistakable primary element?
2. **Right dials?** — Are the emphasis tools appropriate for the content type?
3. **Strong enough?** — Is the difference between primary and secondary dramatic, not subtle?
4. **Not screaming?** — Is emphasis reserved for what matters, or is everything loud?
5. **Goal-driven?** — Does emphasis serve the user's status and action goals?

### Accessibility (A) — Score 1–5

1. **Contrast?** — Does text meet WCAG 2.1 AA minimums?
2. **Target size?** — Are touch/click targets at least 44×44px?
3. **Color independence?** — Does meaning survive without color?
4. **Keyboard nav?** — Can all elements be reached without a mouse?
5. **Forgiving?** — Does the UI work for tired, distracted, or unfamiliar users?

### Reward (R) — Score 1–5

1. **30-second test?** — Does the user receive positive feedback within 30 seconds?
2. **Control?** — Does the user feel in charge (Safety, Certainty, Agency)?
3. **Competence?** — Does the interface confirm the user is doing well (Progress, Achievement)?
4. **Recognition?** — Does the interface acknowledge the user's identity or effort?
5. **Not selfish?** — Do rewards serve the user, not just the business?

## Usage Mechanics

1. **Prioritize** — Fix the lowest pillar first. It's the biggest drag on overall quality.
2. **Align** — Compare scores with colleagues to surface disagreements before debates over taste.
3. **Iterate** — Re-score after each change to verify what actually improved.

**Operating question:** "How can we improve this pillar by 1 point?"
**North star question:** "What would make this a 5?"

## Worked Example: Shield+ Dashboard

### Before Scores (9/25)
| Pillar | Score | Key Issue |
|--------|-------|-----------|
| C | 3/5 | Unnecessary info density, charts without user actions |
| L | 1/5 | No visual grouping, F-pattern broken, no semantic zones |
| E | 1/5 | Everything same visual weight, no clear #1 element |
| A | 2/5 | Small text, tight click targets, color-dependent states |
| R | 2/5 | No positive feedback, no progress indication |

### After Scores (21/25)
| Pillar | Score | Key Issue Remaining |
|--------|-------|-----------|
| C | 5/5 | — (all copy issues resolved) |
| L | 4/5 | Date filters easy to miss |
| E | 5/5 | — (clear hierarchy established) |
| A | 3/5 | Some labels and click targets still small |
| R | 4/5 | "Challenges" copy opaque without setup context |

### Key Redesign Changes
- **Copy:** "Overview" → "Account monitoring" (specific), UUID labels → email addresses (humanized), "Health score: 89%" (reward concretization), evaluations count stat removed (unnecessary)
- **Layout:** Three semantic zones (Detractors / General / Challenges), Gestalt common regions, chart type fixes (donut → pie for proportions, stacked bar for Challenges)
- **Emphasis:** Health score as large teal donut (Visualization dial), zone titles enlarged and rewritten, "Great job!" expanded to conversational prose
- **Cross-pillar moment:** Layout pass surfaced a Copy problem mid-process (chart labels needed rewriting). This is the C.L.E.A.R. loop working as designed.
