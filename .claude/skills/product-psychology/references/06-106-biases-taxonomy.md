# 106 Cognitive Biases & Principles — Taxonomy

From the Growth.Design Product Psychology Masterclass master reference. Organized into four categories that map to a sequential user arc: Noticing → Making Sense → Acting Over Time → Remembering.

Use this reference when you need to identify which cognitive bias is at play in a specific product situation, or when you want to explore biases beyond the core set covered in the skill flow.

---

## Category 1: INFORMATION (How Users Perceive & Process)

How users filter, notice, and take in what's in front of them. Maps primarily to **Skill 4 (Block)**.

Key principles in this category:
- Hick's Law — decision time increases with number of choices
- Selective Attention — goal-irrelevant info is filtered out
- Banner Blindness — learned ignoring of ad-positioned content
- Edge Blindness — content at viewport edges gets missed
- Priming — prior exposure shapes subsequent perception
- Baader-Meinhof Effect — recently noticed things seem everywhere
- Von Restorff Effect (Isolation Effect) — distinctive items are remembered better
- Pattern Recognition — the brain seeks and matches familiar patterns
- Change Blindness — gradual changes go unnoticed
- Inattentional Blindness — unexpected objects invisible when focused
- Serial Position Effect — first and last items in a series are remembered best
- Attentional Bias — perception biased toward emotionally salient stimuli
- Sensory Adaptation — constant stimuli fade from awareness
- Mere Exposure Effect — familiarity breeds preference

## Category 2: MEANING (How Users Interpret & Assign Value)

How users make sense of what they perceive and assign value to it. Maps primarily to **Skill 5 (Interpret)**.

Key principles in this category:
- Anchoring — first reference point sets expectations
- Framing Effect — same info, different presentation, different interpretation
- Loss Aversion — losses weigh ~2× more than equivalent gains
- Endowment Effect — ownership increases perceived value
- IKEA Effect — self-made = more valued
- Cognitive Load — finite processing capacity
- Miller's Law — 7±2 working memory limit
- Familiarity / Jakob's Law — users expect your site to work like others
- Confirmation Bias — seeking info that confirms existing beliefs
- Dunning-Kruger Effect — overestimating ability in areas of low competence
- Curse of Knowledge — experts forget what it's like to not know
- Ambiguity Effect — avoiding options with unknown probability
- Status Quo Bias — preference for current state
- Peak-End Rule — experiences judged by peak intensity and ending
- Halo Effect — one positive trait influences overall judgment
- Affect Heuristic — emotions drive decisions, then rationalization follows
- Authority Bias — deference to perceived experts
- Social Proof — following others' behavior under uncertainty
- Bandwagon Effect — adoption increases with perceived popularity
- Contrast Effect — perception altered by comparison to recent stimuli

## Category 3: TIME (How Behavior Shifts Across Decisions)

How user behavior changes across time, repeated interactions, and sequential decisions. Maps primarily to **Skill 6 (Act)** and **Skill 9 (Journey Improve)**.

Key principles in this category:
- Hyperbolic Discounting — immediate rewards disproportionately preferred
- Commitment and Consistency — past actions constrain future choices
- Sunk Cost Fallacy — past investment drives irrational continuation
- Goal Gradient Effect — effort increases near completion
- Zeigarnik Effect — incomplete tasks create persistent tension
- Scarcity — limited availability increases perceived value
- Urgency — time pressure accelerates decisions
- Reactance — restricted freedom increases desire for restricted option
- Decision Fatigue — decision quality degrades over time
- Default Effect — pre-selected options disproportionately chosen
- Progressive Disclosure — complexity revealed gradually
- Fresh Start Effect — temporal landmarks increase motivation
- Planning Fallacy — underestimating time for tasks
- Temporal Discounting — future costs/benefits feel less real
- Present Bias — current state weighted more heavily than future
- Curiosity Gap — information gap drives engagement
- Variable Reward — unpredictable rewards sustain engagement longer

---

## Category 4: MEMORY (How Experiences Get Encoded & Retrieved)

How users remember, forget, and reconstruct experiences. Maps primarily to **Skill 7 (Store)**.

Key principles in this category:
- Peak-End Rule — judged by peak intensity and final moment
- Recency Effect — latest information weighted most heavily
- Primacy Effect — first information anchors subsequent processing
- Availability Heuristic — easily recalled = perceived as more common
- Spacing Effect — distributed exposure aids retention
- Testing Effect — retrieval practice strengthens memory
- Generation Effect — self-generated info remembered better
- Picture Superiority Effect — images encoded better than words
- Bizarreness Effect — unusual items remembered better
- Nostalgia Effect — past experiences recalled more positively
- Rosy Retrospection — past remembered more favorably than experienced
- Mood Congruence — current mood biases memory retrieval
- Source Confusion — misattribution of where information came from
- Misinformation Effect — post-event information alters memory
- Cross-Race Effect — better recognition of own-race faces
